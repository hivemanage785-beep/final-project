================================================================================
                    BUZZ-OFF VERCEL DEPLOYMENT GUIDE
================================================================================

This guide outlines how to deploy the Buzz-Off full-stack application to Vercel.

PREREQUISITES:
--------------
1. A Vercel Account (https://vercel.com)
2. Vercel CLI installed (optional, but recommended: `npm i -g vercel`)
3. A MongoDB Atlas cluster (Vercel does not host local databases)
4. A Firebase Project for authentication.

DEPLOYMENT STRATEGY:
--------------------
Because this is a multi-service monorepo, you must deploy each service 
individually for the best performance and compatibility.

--- STEP 1: DEPLOY ML SERVICE (FASTAPI) ---
1. Navigate to: /final-project-dev-jo/ml-service
2. Ensure 'requirements.txt' and 'app.py' are in the root of this folder.
3. If using Vercel CLI, run: `vercel` inside the ml-service folder.
4. Set the project name to 'buzzoff-ml-service'.
5. Once deployed, note the URL (e.g., https://buzzoff-ml-service.vercel.app).

--- STEP 2: DEPLOY BACKEND API (NODE/EXPRESS) ---
1. Navigate to: /final-project-dev-jo/backend
2. Ensure 'vercel.json' exists (I have already created this for you).
3. If using Vercel CLI, run: `vercel` inside the backend folder.
4. Set the project name to 'buzzoff-backend'.
5. IMPORTANT: Add environment variables in Vercel Settings:
   - MONGO_URI: <Your MongoDB Atlas URI>
   - ML_SERVICE_URL: <URL from Step 1>
   - JWT_SECRET: <A long random string>
   - NODE_ENV: production
6. Once deployed, note the URL (e.g., https://buzzoff-backend.vercel.app).

--- STEP 3: DEPLOY FRONTEND UI (REACT/VITE) ---
1. Navigate to: /final-project-dev-jo/frontend
2. If using Vercel CLI, run: `vercel` inside the frontend folder.
3. Set the project name to 'buzzoff-frontend'.
4. IMPORTANT: Add environment variables for API connection:
   - VITE_API_BASE_URL: <URL from Step 2>
   - VITE_FIREBASE_API_KEY: <From Firebase Console>
   - VITE_FIREBASE_AUTH_DOMAIN: <From Firebase Console>
   - VITE_FIREBASE_PROJECT_ID: <From Firebase Console>
   - VITE_GEMINI_API_KEY: <From AI Studio>
5. Deploy and start!

TROUBLESHOOTING:
----------------
- DATABASE: If the backend fails to start, ensure your MongoDB Atlas IP White-list 
  allows connections from anywhere (0.0.0.0/0) so Vercel can connect.
- CORS: If the frontend cannot communicate with the backend, update the 
  FRONTEND_URL in your backend environment variables to match your final 
  Vercel frontend URL.
